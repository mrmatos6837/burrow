<task type="auto">
  <name>Task 1: Create CLI router with all 8 subcommands</name>
  <files>.claude/burrow/burrow-tools.cjs</files>
  <action>
Create `.claude/burrow/burrow-tools.cjs` as a CLI entry point. Add `#!/usr/bin/env node` shebang and `'use strict'`.

The file structure:
1. Require core, storage, tree from ./lib/
2. `main()` function wrapped in try/catch -- catch outputs `{success: false, error: e.message, code: "STORAGE_ERROR"}` via core.errorOut
3. Extract command from `process.argv[2]`, subArgs from `process.argv.slice(3)`
4. Call `ensureDataDir(process.cwd())` before any command
5. Switch on command, each case:
   - Parses subArgs with `util.parseArgs()` (define options per command)
   - Loads data via `storage.load(cwd)`
   - Calls appropriate tree function
   - For mutations (add, edit, delete, move): calls `storage.save(cwd, data)` after tree operation
   - Outputs result via `core.output({success: true, data: result})`

Subcommand implementations:

**add**: parseArgs with options: title (string, required), parent (string), notes (string, default ''), position (string). Call `tree.addItem(data, {title, parentId: values.parent || null, notes: values.notes, position: values.position != null ? parseInt(values.position, 10) : undefined})`. Save and output created item.

**edit**: First positional is the ID. parseArgs with options: title (string), notes (string), ordering (string). Validate ordering is one of custom/alpha-asc/alpha-desc if provided. Call `tree.editItem(data, id, {title, notes, ordering})`. Save and output updated item.

**delete**: First positional is the ID. No additional flags. Call `tree.deleteItem(data, id)`. Save and output result.

**move**: First positional is the ID. parseArgs with options: parent (string), position (string). If --parent is empty string or "root", set newParentId to null. Call `tree.moveItem(data, id, newParentId, position != null ? parseInt(position, 10) : undefined)`. If moveItem returns an error object (has success:false), output that error. Otherwise save and output moved item.

**get**: First positional is the ID. Call `tree.findById(data, id)`. If null, errorOut with NOT_FOUND. Output the full item (including all nested children).

**children**: First positional is the ID. Call `tree.getChildren(data, id)`. Output the ordered children array.

**list**: Optional first positional as parentId. Call `tree.listItems(data, parentId || null)`. Output the ordered items array.

**path**: First positional is the ID. Call `tree.getPath(data, id)`. If null, errorOut with NOT_FOUND. Output the ancestry array (each element is {id, title} -- strip children to keep output clean).

**default**: errorOut with `Unknown command: ${command}. Available: add, edit, delete, move, get, children, list, path` and code INVALID_OPERATION.

No --help flag needed in Phase 1. Keep the router thin -- all logic lives in tree.cjs.

Handle the "move returns error" case: tree.moveItem may return `{success: false, error: "..."}` for cycle detection. Check if result has `success === false` and output it directly (do not save).

  <action>
Create `.claude/burrow/test/cli.test.cjs` using `node:test` and `node:assert`.

Create a helper function `run(args, cwd)` that:
- Calls `child_process.execFileSync('node', [path.resolve(__dirname, '..', 'burrow-tools.cjs'), ...args], {cwd, encoding: 'utf-8', env: {...process.env}})`
- Parses stdout as JSON and returns it
- On error (exit code 1), catches the error, parses stdout from the error object, and returns the parsed JSON

Each test group uses its own temp directory via `fs.mkdtempSync(path.join(os.tmpdir(), 'burrow-cli-'))` to isolate tests. Clean up temp dirs in afterEach/after.

- it('adds item with --position'): add two items, third with --position 0, verify it gets position 0

**describe('edit')**:

- it('edits ordering'): add item, edit with --ordering alpha-asc, verify data.children.ordering === "alpha-asc"

**describe('delete')**:

- it('reports descendant count'): add parent with 2 children, delete parent, verify descendantCount === 2

**describe('move')**:
- it('moves item to different parent'): add A and B at root, move A to child of B, verify via get
- it('moves item to root'): add parent, add child, move child to root (--parent ""), verify via list
- it('rejects cycle'): add A, add B as child of A, try move A to child of B, verify error

**describe('get')**:
