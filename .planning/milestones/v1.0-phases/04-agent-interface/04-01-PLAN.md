  <action>
Create the workflow file at `.claude/burrow/workflows/burrow.md`. This file defines all agent behavior for Burrow interactions. It is loaded only by the `/burrow` NL command (shortcuts do NOT load it).

Structure the workflow with these required sections (organization and depth is at your discretion):

1. **Overview** -- Brief description: Burrow is an infinitely nestable card tool. The agent interprets natural language, calls CLI operations, and presents results. CLI path: `node .claude/burrow/burrow-tools.cjs`.

2. **Invariants (NEVER violate)** -- Exactly these 6 rules, no more:
   1. Never delete without confirmation -- always ask "Delete 'X' and its N children?" before running delete
   2. Never modify data without explicit user request -- read operations are safe; write operations require clear user intent
   3. Always clarify ambiguous card references -- if a title matches multiple cards, list them and ask which one
   4. Always use tool output for rendering -- run CLI without --json flag; output the result directly to the user. Do not wrap in code blocks. Do not add headers or formatting
   5. Never show raw JSON unless the user asks for it -- use --json only for internal data parsing, never for user-facing output
   6. Read before write -- ingest tree state (via `get --depth 0 --json`) before mutations to resolve card references correctly

3. **Command reference** -- Table or list of all CLI commands the agent can call:
   - `get [id] [--depth N] [--full] [--include-archived] [--archived-only]` -- universal view command
   - `add --title "..." [--parent <id>] [--body "..."]` -- create a card
   - `edit <id> [--title "..."] [--body "..."]` -- modify a card
   - `delete <id>` -- delete card and descendants (requires confirmation)
   - `move <id> --to <parent-id>` -- move card to new parent
   - `archive <id>` -- archive card and descendants
   - `unarchive <id>` -- restore card and descendants
   - `dump` -- alias for `get --depth 0` (full tree)
   - `path <id>` -- show ancestry from root to card
   All commands invoked as: `node .claude/burrow/burrow-tools.cjs <command> [args]`

4. **Rendering rules** -- Agent runs CLI without --json and passes rendered output straight to the user. Agent never re-renders or reformats tool output. For multi-step operations: write a plain English summary of changes, then run one or more `get` commands to show end state from best perspective. Default depth is 1 (card + direct children).

5. **NL intent parsing** -- How to interpret natural language:
   - Card reference resolution: run `get --depth 0 --json` to ingest tree, fuzzy match titles, clarify on multiple matches
   - Missing card handling: offer to create
   - Multi-step operations: chain CLI calls, then summarize + show end state
   - Bulk operations: confirm before executing
   - When showing what command will be run, use the format: "Running: `burrow <command> <args>`"
   - Default behavior with no args: show root view at depth 1 (`get`)

6. **Worked examples** -- Include 3-5 grounded examples covering:
   - Simple card creation via NL ("add a card called X under Y")
   - Viewing/navigating ("show me the bugs section", "what's in the project?")
   - Multi-step operation ("move all the auth cards under security")
   - Ambiguity resolution ("archive the login card" when multiple exist)
   Each example should show: user input, agent thought process, commands run, output presentation.

7. **Tone and style** -- Functional and terse. "Added 'OAuth bug' under bugs." No personality, no burrow metaphors, no excessive commentary.

Keep the file focused and practical. This is a reference the agent loads into context -- every unnecessary word costs tokens.

  <action>
Create the main NL parser command file at `.claude/commands/burrow/burrow.md`.

This is the ONLY command that loads the full workflow. It accepts natural language and interprets user intent.

Use this structure (based on existing GSD command format):

```markdown
---
name: burrow
description: Interact with Burrow using natural language -- add, view, move, archive cards
argument-hint: "[natural language request, or empty for root view]"
allowed-tools:
  - Bash
  - Read
---
<objective>
Interpret the user's natural language request and perform the corresponding Burrow operations.
With no arguments, show the root view at depth 1.
</objective>

<execution_context>
@./.claude/burrow/workflows/burrow.md
</execution_context>

<context>
User request: $ARGUMENTS
CLI path: node .claude/burrow/burrow-tools.cjs
</context>

<process>
Follow the workflow instructions to:
1. Parse the user's intent from $ARGUMENTS
2. If empty, run: node .claude/burrow/burrow-tools.cjs get
3. If a data operation, read tree state first (--json) to resolve references
4. Show what command you're running (transparency)
5. Run the CLI command(s) without --json
6. Output the result directly -- do not reformat
</process>
```

Important: The `name: burrow` (without colon) in frontmatter should register this as `/burrow`. If Claude Code treats `commands/burrow/burrow.md` as `/burrow:burrow`, the fix would be moving this to `commands/burrow.md` -- but try the directory approach first since it matches the GSD pattern (`commands/gsd/` directory with command files).

Note: The `allowed-tools` list includes `Read` because the agent may need to read files for context, and `Bash` for running CLI commands.
