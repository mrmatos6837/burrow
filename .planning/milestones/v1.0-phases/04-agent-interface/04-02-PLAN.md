  <action>
Create 9 slash command files in `.claude/commands/burrow/`. All shortcuts are thin wrappers -- they do NOT load the workflow file. They parse `$ARGUMENTS` and shell out to `burrow-tools.cjs`.

Each shortcut file follows this pattern:

```markdown
---
name: burrow:COMMAND
description: Brief description
argument-hint: "flags and args"
allowed-tools:
  - Bash
---
Brief one-line instruction.

Run: `node .claude/burrow/burrow-tools.cjs COMMAND $ARGUMENTS`

Output the CLI result directly. Do not reformat or wrap in code blocks.
```

Create these files with the exact CLI flag mappings:

1. **add.md** (`/burrow:add`)
   - argument-hint: `--title "card title" [--parent <id>] [--body "content"]`
   - Maps to: `burrow-tools.cjs add $ARGUMENTS`

2. **show.md** (`/burrow:show`)
   - argument-hint: `[<id>] [--depth N] [--full] [--include-archived] [--archived-only]`
   - Maps to: `burrow-tools.cjs get $ARGUMENTS` (note: show maps to CLI `get`)

3. **edit.md** (`/burrow:edit`)
   - argument-hint: `<id> [--title "new title"] [--body "new body"]`
   - Maps to: `burrow-tools.cjs edit $ARGUMENTS`

4. **move.md** (`/burrow:move`)
   - argument-hint: `<id> --to <parent-id>`
   - Maps to: `burrow-tools.cjs move $ARGUMENTS`

5. **delete.md** (`/burrow:delete`)
   - argument-hint: `<id>`
   - Maps to: `burrow-tools.cjs delete $ARGUMENTS`
   - Include instruction: "Ask the user to confirm before running the delete command. Show what will be deleted."

6. **archive.md** (`/burrow:archive`)
   - argument-hint: `<id>`
   - Maps to: `burrow-tools.cjs archive $ARGUMENTS`

7. **unarchive.md** (`/burrow:unarchive`)
   - argument-hint: `<id>`
   - Maps to: `burrow-tools.cjs unarchive $ARGUMENTS`

8. **dump.md** (`/burrow:dump`)
   - argument-hint: "" (no args)
   - Maps to: `burrow-tools.cjs dump` (alias for `get --depth 0`)

9. **help.md** (`/burrow:help`)
   - argument-hint: "" (no args)
   - Does NOT call CLI. Instead, outputs a formatted command reference showing:
     - `/burrow [request]` -- natural language (main entry point)
     - `/burrow:show [id] [--depth N]` -- view cards
     - `/burrow:add --title "..." [--parent id]` -- add card
     - `/burrow:edit id [--title "..."] [--body "..."]` -- edit card
     - `/burrow:move id --to parent-id` -- move card
     - `/burrow:delete id` -- delete card (with confirmation)
     - `/burrow:archive id` -- archive card
     - `/burrow:unarchive id` -- restore card
     - `/burrow:dump` -- full tree view
   - Keep it terse. Just the command table, maybe 2-3 usage examples.
   - allowed-tools: none needed (just outputs text)

Key rules for all shortcuts:
- Do NOT include `@` references to the workflow file
- Do NOT include `Read` in allowed-tools (shortcuts don't need to read files)
- Exception: delete.md should include a confirmation step before executing
- The `show` command maps to CLI `get` (user-facing name differs from CLI name)
