  <action>
Create `.claude/burrow/lib/render.cjs` as a CommonJS module exporting four functions: `renderCard()`, `renderMutation()`, `renderPath()`, `renderError()`. All functions are pure -- they take data and return strings. No stdout writes, no process.exit, no file I/O.

**Internal helpers (not exported):**
- `formatAge(isoString)` -- relative timestamp using thresholds: <60s="just now", <60min="Xm ago", <24h="Xh ago", <7d="Xd ago", <52w="Xw ago", else "Xy ago"
- `formatCreatedDate(isoString)` -- "YYYY-MM-DD (Xd ago)" format matching CONTEXT.md examples
- `formatBreadcrumb(ancestors, cardTitle, context)` -- "burrow > ancestors > card name" with optional context after dot separator
- `formatCardLine(card, prefix, termWidth)` -- single card line in tree listing: `[8-char ID] title dot  age  (N)`. Uses body dot marker, right-aligned age via two-pass approach. Truncates long titles with ellipsis.
- `HR` constant -- 40x horizontal bar character

**Exported functions:**

`renderCard(card, breadcrumbs, opts)`:
- card: full card object {id, title, created, archived, body, children}
- breadcrumbs: array of {id, title} ancestors (empty array for root)
- opts: {full: boolean, termWidth: number, archiveFilter: string}
- Section order: breadcrumb header > empty line > HR > title > HR > metadata (id, created, archived) > HR > children > HR > body > HR
- Children section: "children: N cards (M total)" header when children exist, then tree lines with box-drawing. Show "(none)" when empty.
- Body section: show content indented 2 spaces, or "(empty)". Cap at ~200 chars with truncation notice unless `full=true`. Body lines get newlines replaced with newline + 2-space indent.
- When archiveFilter is "include-archived", show [archived] label on archived children. When "active" (default), filter out archived children from display. When "archived-only", show only archived children.
- For root card: accept id="(root)", title="burrow".
- Children count: count active children for "N cards", use countActiveDescendants-style counting for "(M total)".

`renderMutation(type, result, opts)`:
- type: 'add'|'edit'|'delete'|'move'|'archive'|'unarchive'
- result: command-specific result object
- opts: {breadcrumbs, card, oldTitle, oldBody, fromParentTitle, termWidth}
- add: "checkmark Added card\n\n" + full card detail via renderCard
- edit: "checkmark Edited card\n" + diff lines (title/body old->new, capped at ~40 chars with ellipsis) + "\n" + full card detail
- delete: "checkmark Deleted \"title\" [id] (and N children)" -- one-liner. Omit "(and 0 children)" when no children.
- move: "checkmark Moved \"title\" [id]: source -> target" -- one-liner
- archive: "checkmark Archived \"title\" [id] (and N children)" -- one-liner. Omit "(and 0 children)".
- unarchive: "checkmark Unarchived \"title\" [id] (and N children)" -- one-liner. Omit "(and 0 children)".

`renderPath(pathArray)`:
- pathArray: [{id, title}, ...] from root to target
- Returns: "burrow > title1 > title2 > ... > titleN [lastId]"

`renderError(message)`:
- Returns: "cross-mark message" (using Unicode cross-mark symbol)

Use symbols: checkmark for success, cross-mark for errors, branch/corner for tree lines, dot for body indicator, right-angle bracket for breadcrumb separator. Plain text only -- no ANSI color codes per locked decision.
