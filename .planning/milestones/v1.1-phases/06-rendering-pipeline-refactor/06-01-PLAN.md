  <action>
Rewrite the `renderTree` function in mongoose.cjs to return nested card objects instead of a flat array with depth properties.

**New renderTree behavior:**
1. Keep the same signature: `renderTree(data, rootId, opts)` with `opts: {depth, archiveFilter}`
2. Keep `maxDepth` logic: `depth === 0 ? Infinity : (depth ?? 1)`
3. Keep `shouldInclude` archive filter logic (identical to current)
4. Keep breadcrumb computation (identical to current)
5. CHANGE: Instead of pushing to `flatArray` with depth, build nested `children` arrays

**New internal walk function:**
```javascript
function buildNested(cards, currentDepth) {
  const result = [];
  for (const card of cards) {
    if (shouldInclude(card)) {
      const activeDesc = countActiveDescendants(card); // once per card
      const entry = {
        id: card.id,
        title: card.title,
        descendantCount: activeDesc,
        hasBody: !!(card.body && card.body.trim()),
        bodyPreview: makePreview(card.body),
        created: card.created,
        archived: card.archived,
        children: (currentDepth < maxDepth && card.children && card.children.length)
          ? buildNested(card.children, currentDepth + 1)
          : [],
      };
      result.push(entry);
    }
  }
  return result;
}
```

**Return shape stays the same key names:** `{ breadcrumbs, cards }` — but `cards` is now nested tree, not flat array. No `depth` property on cards (depth is implicit from nesting).

**Do NOT change:** `countActiveDescendants`, `makePreview`, `getPath`, `findById` — they stay as-is. Only `renderTree` changes.

**Do NOT remove:** the flat `depth` property silently — it should simply not be in the new output. The `nestFlatCards` consumer in burrow-tools.cjs will be updated in Plan 02.
