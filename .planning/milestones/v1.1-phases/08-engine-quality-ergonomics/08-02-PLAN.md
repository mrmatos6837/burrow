   - `data.cards` or `data.items` is an array (v1 used items, v2 uses cards)

<task type="auto" tdd="true">
  <name>Task 2: makePreview truncate-first + eliminate collectAllIds on addCard</name>
  <files>.claude/burrow/lib/mongoose.cjs, .claude/burrow/lib/core.cjs, test/mongoose.test.cjs</files>
  <behavior>
    - makePreview on a 10000-char body with newlines processes at most ~83 characters (truncate length + ellipsis)
    - makePreview("short\nbody") returns "short body" (newlines replaced, no truncation)
    - makePreview on 80-char body with newlines returns full cleaned string
    - addCard generates unique IDs without calling collectAllIds
    - addCard on a tree with 1000 cards does not walk all 1000 cards to generate an ID
    - generateId() works without an existingIds parameter (uses crypto.randomUUID collision probability)
  </behavior>
  <action>
1. **makePreview truncate-first (PERF-08):** Current code: `body.replace(/\n/g, ' ')` on full body, then truncate. Reverse order: `body.slice(0, 83).replace(/\n/g, ' ')` then truncate at 80 with '...'. Use 83 as slice length (80 + 3 for potential '...' suffix). This avoids processing newlines in potentially huge body strings that will be thrown away anyway.

2. **Eliminate O(n) collectAllIds (PERF-10):** The 8-char hex ID space is 16^8 = 4.3 billion values. crypto.randomUUID produces cryptographically random values. The probability of collision with even 10,000 cards is vanishingly small (~0.000001%). Change approach:
   - Remove `collectAllIds` import from mongoose.cjs
   - Change `generateId` in core.cjs to take no required parameter: `function generateId()` — just generate `crypto.randomUUID().replace(/-/g, '').slice(0, 8)` and return it. No collision check needed.
   - Remove the `collectAllIds` function from core.cjs exports entirely (it's only used by addCard via generateId)
   - Update addCard to call `generateId()` with no arguments
   - If any other code imports collectAllIds, keep the export but mark with JSDoc as deprecated

3. **Verify no other callers of collectAllIds:** Check that no other file imports collectAllIds. If the test files use it, update them.

4. **Add/update tests:** Test makePreview with very large body strings (verify truncation happens before newline replacement by checking performance — or just verify output correctness). Test that addCard still generates unique IDs across multiple calls.
