<task type="auto" tdd="true">
  <name>Task 1: Consolidate count functions, fix findParent, refactor moveCard</name>
  <files>.claude/burrow/lib/mongoose.cjs, test/mongoose.test.cjs</files>
  <behavior>
    - countDescendants(card) returns total descendants (current behavior preserved)
    - countDescendants(card, { activeOnly: true }) returns non-archived descendants (replaces countActiveDescendants)
    - countDescendants(card, { activeOnly: false }) or countDescendants(card) counts all descendants
    - findParent single-traversal: searches root level and nested in one recursive walk (no separate root loop + nested loop)
    - moveCard: combines findById + getPath (cycle check) + findParent (remove) into at most 2 walks — one walk that finds the card AND checks ancestry, one getContainer for target
    - All existing mongoose.test.cjs tests still pass
  </behavior>
  <action>
1. **countDescendants consolidation (PERF-02):** Add optional second parameter `opts = {}` with `activeOnly` boolean. When `activeOnly` is true, skip archived children and their subtrees (current countActiveDescendants behavior). Delete the standalone countActiveDescendants function. Update all internal callers: renderTree's buildNested calls countDescendants(card, { activeOnly: true }), archiveCard/unarchiveCard call countDescendants(card) (all descendants). Export countDescendants only (remove countActiveDescendants from exports). Keep backward compat: bare countDescendants(card) counts all.

2. **findParent single traversal (QUAL-02):** Rewrite findParent to use a single recursive function that checks both root-level (parent=null, container=data.cards) and nested cards in one pass. Current code has a separate for-loop over data.cards at root level, then a nested `search` function that loops over data.cards again. Combine into one recursive search that handles both cases.

3. **moveCard optimization (PERF-01):** Currently moveCard does: findById (walk 1), getPath for cycle check (walk 2), findParent for source removal (walk 3), getContainer for target (walk 4). Refactor to: (a) a single `findCardWithAncestry(data, cardId)` internal helper that returns `{card, parent, container, ancestors}` in one walk, (b) check cycle by testing if newParentId is in ancestors, (c) use the already-found container to splice, (d) getContainer for target insertion. This reduces 4 walks to 2.

4. **Update tests:** Add tests for countDescendants with activeOnly option. Verify findParent still returns correct {parent, container} for root and nested cards. Verify moveCard still handles all edge cases (cycle detection, root-to-nested, nested-to-root, reorder).
  </action>
  <verify>
    <automated>node --test test/mongoose.test.cjs</automated>
  </verify>
  <done>countDescendants is parameterized (activeOnly option), countActiveDescendants is removed, findParent uses single traversal, moveCard uses at most 2 tree walks. All tests pass.</done>
</task>
