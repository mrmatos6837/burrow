<task type="auto">
  <name>Task 1: Create /burrow:update slash command</name>
  <files>.claude/commands/burrow/update.md</files>
  <action>
    Create `.claude/commands/burrow/update.md` — this is the slash command that Claude Code exposes as `/burrow:update`.

    The slash command should instruct Claude to:
    1. Determine the burrow source directory (the repo where burrow's source code lives). This is the directory containing `install.cjs` at its root. Since the slash command runs from within the user's project, the source dir needs to be known. For local (pre-npm) installs, the source is the original cloned burrow repo.

    IMPORTANT: The slash command needs to know WHERE the burrow source repo is. For pre-npm local installs, the user cloned the burrow repo somewhere and ran `node /path/to/burrow/install.cjs`. The installer could record the source path in a metadata file during install. But for simplicity, the slash command should:
    - Check if a `.claude/burrow/.source-dir` file exists (written by installer during install/upgrade)
    - If it exists, use that path as sourceDir
    - If not, ask the user for the path to their burrow repo clone

    The command content should be a prompt that tells Claude:
    ```
    Run the burrow installer in upgrade mode to update burrow to the latest version.

    Steps:
    1. Read `.claude/burrow/.source-dir` to find the burrow source repository path
    2. If the file doesn't exist, ask the user where their burrow repo is cloned
    3. Run: node <sourceDir>/install.cjs --yes
    4. Report the result to the user
    ```

    Format: Follow the same markdown structure as other slash commands in `.claude/commands/burrow/` (look at help.md, read.md etc for style reference).
  </action>

<task type="auto">
  <name>Task 2: Update install.cjs to write version cache and source-dir breadcrumb after install/upgrade</name>
  <files>install.cjs</files>
  <action>
    Modify install.cjs to do two things after a successful install or upgrade:

    1. **Write `.claude/burrow/.source-dir`** in the target directory — contains the absolute path to the source repo (the directory where install.cjs lives: `__dirname` or `path.resolve('.')`). This lets `/burrow:update` find the source repo later.
       - Write after performInstall or performUpgrade succeeds
       - Content: just the absolute path string + newline
       - On upgrade, overwrite (source location may change if user re-clones)

    2. **Write `.planning/burrow/.update-check`** cache in the target directory — contains JSON with `{ lastCheck, sourceVersion, installedVersion }`. This is what burrow-tools.cjs reads for the passive notification (Plan 01).
       - Import `getSourceVersion` from `./.claude/burrow/lib/version.cjs` (require at top)
       - After performInstall/performUpgrade, read source version via `getSourceVersion(sourceDir)` and installed version via reading the VERSION file that was just copied to target
       - Write cache: `{ lastCheck: new Date().toISOString(), sourceVersion: <string>, installedVersion: <string> }`
       - Wrap in try/catch — cache writing must never fail the install

    Where to add in install.cjs:
    - After the `performInstall(sourceDir, targetDir)` call in the fresh install flow
    - After the `performUpgrade(sourceDir, targetDir)` call in the upgrade flow
    - NOT after repair (repair doesn't change versions)

    Also add `.claude/burrow/.source-dir` and `.planning/burrow/.update-check` to the `.gitignore` entry logic (these are local-only files that should not be committed). Use the existing `ensureGitignoreEntry` pattern from the installer engine, or simply add the entries alongside the existing `.planning/burrow/cards.json.bak` gitignore handling.

    WAIT: install.cjs does not have direct access to ensureGitignoreEntry (it's internal to installer.cjs). Instead, just write the files — they are dotfiles in directories that are already partially gitignored. Add a note as a code comment that these are local-only breadcrumbs.
