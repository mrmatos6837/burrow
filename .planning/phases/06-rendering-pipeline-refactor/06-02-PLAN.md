  <action>
Update burrow-tools.cjs to use the new nested renderTree output directly:

**1. Delete nestFlatCards function** (lines 51-76). Remove entirely.

**2. Update `read` command (card view, lines 299-326):**

CURRENT flow:
```
card = findById(data, id)
breadcrumbs = getBreadcrumbs(data, id)  // separate tree walk
treeResult = renderTree(data, id, {depth, archiveFilter})  // returns flat
childCards = nestFlatCards(treeResult.cards.filter(c => c.depth >= 1))  // reconstruct nesting
cardCopy = {...card, children: childCards}
renderCard(cardCopy, breadcrumbs, opts)
```

NEW flow:
```
treeResult = renderTree(data, id, {depth, archiveFilter})
if (!treeResult) handleError(`Card not found: ${id}`)
// treeResult.cards[0] IS the root card with nested children already
// treeResult.breadcrumbs IS the breadcrumbs already computed
const cardToRender = treeResult.cards[0]
// Merge original card's body for full display (renderTree only has bodyPreview)
const fullCard = tree.findById(data, id)
cardToRender.body = fullCard.body  // renderCard needs the full body
cardToRender.title = fullCard.title  // use original title (not truncated)
renderCard(cardToRender, treeResult.breadcrumbs || [], opts)
```

Simplify the two branches (treeResult.cards.length > 1 vs else) into one path — renderTree always returns the card with its nested children now. No branching needed.

**3. Update `read` command (root view, lines 327-345):**

CURRENT flow:
```
treeResult = renderTree(data, null, {depth, archiveFilter})
prunedChildren = nestFlatCards(treeResult.cards, 0)  // reconstruct nesting
rootCard = {id: '(root)', title: 'burrow', ..., children: prunedChildren}
renderCard(rootCard, [], opts)
```

NEW flow:
```
treeResult = renderTree(data, null, {depth, archiveFilter})
rootCard = {
  id: '(root)',
  title: 'burrow',
  created: data.cards[0]?.created || new Date().toISOString(),
  archived: false,
  body: '',
  children: treeResult.cards,  // already nested!
}
renderCard(rootCard, [], opts)
```

**4. Update `dump` command (lines 349-386):**

Same pattern as root view — replace `nestFlatCards(treeResult.cards, 0)` with `treeResult.cards` directly.

**5. Consolidate getBreadcrumbs:**

For `add` and `edit` commands that still call `getBreadcrumbs(data, id)`, keep the function but note it remains a separate `getPath` call. This is acceptable because `add` and `edit` do NOT call `renderTree` — they only need breadcrumbs for mutation rendering. The REND-02 duplication was specifically about `read`/`dump` calling both `renderTree` (which internally computes breadcrumbs) AND `getBreadcrumbs` separately.

For `read` command: replace `getBreadcrumbs(data, id)` with `treeResult.breadcrumbs`.

**Key constraint:** The CLI output for `read` and `dump` must be IDENTICAL before and after this change. Run existing CLI tests to verify.
