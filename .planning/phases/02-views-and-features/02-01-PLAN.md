  <behavior>
    - Migration converts v1 data to v2: root `items` key -> `cards`, drops `ordering` at root, drops `position` from cards, renames `notes` -> `body`, flattens `children` from `{ordering, cards:[]}` to plain `[]`, sets `version: 2`
    - Migration is idempotent: v2 data passes through unchanged
    - Migration handles edge cases: empty children, children with `items` key instead of `cards`
    - load() returns v2 format regardless of what is on disk
    - save() always writes v2 format (no migration on save)
    - Default empty data structure is `{ version: 2, cards: [] }` (no ordering)
    - addCard creates `{id, title, created, archived: false, body: '', children: []}` -- no position, no ordering in children
    - editCard accepts `{title, body}` -- no ordering param
    - deleteCard works without recompact
    - moveCard splices from source array and inserts at target (end by default, or at `requestedPosition` index) -- no position field assignment, no recompact
    - findById, findParent, getPath, getContainer, countDescendants, collectAllIds all traverse via `card.children` (plain array) not `card.children.cards`
    - getContainer returns `data` for root (with `.cards` array), or the card itself for parentId (with `.children` array) -- but the contract changes: callers use `container.cards` for root or `container.children` for cards. SIMPLER: make getContainer always return `{cards: [...]}` shape -- for root it is `data` (has `.cards`), for a parent card, return a wrapper or just have callers use `findById(parentId).children` directly. Research pattern: getContainer(data, parentId) returns the ARRAY to push into. Simplify: for root return `data.cards`, for parentId return `findById(data, parentId).children`. Return the array itself.
    - ACTUALLY: Simplest approach -- `getContainer` returns the card array to operate on. Root = `data.cards`. ParentId = `findById(data, parentId).children`. Then addCard does `container.push(newCard)`. deleteCard does `container.splice(idx, 1)`. This avoids the `container.cards` pattern entirely.
    - listCards(data, parentId) just returns the card array (root or children) -- no ordering sort
    - getChildren(data, id) returns `card.children` directly -- no ordered sort
    - recompact() and getOrderedChildren() are REMOVED from exports and code

       - Set `data.version = 2`
       - Call `migrate()` in `load()` before returning

    3. **mongoose.cjs**: Major simplification:
       - `findById`: search `card.children` not `card.children.cards`
       - `findParent`: search `card.children` not `children.cards`; return value changes -- `container` is now the plain array the card lives in, or rethink: return `{parent, siblings}` where siblings is the array. Keep backward compat shape by having container be `{cards: array}` wrapper? NO -- simplify. `findParent` returns `{parent: card|null, container: array}` where container is the array the card is in (data.cards for root, parent.children for nested).
       - `getContainer(data, parentId)`: Returns the ARRAY to push into. `null/undefined` -> `data.cards`. Otherwise -> `findById(data, parentId)?.children || null`.
       - `getPath`: traverse `card.children` not `card.children.cards`
       - `addCard(data, {title, parentId, body})`: No position. Creates `{id, title, created: new Date().toISOString(), archived: false, body: body || '', children: []}`. Push onto container array.
       - `editCard(data, id, {title, body})`: No ordering param. Set `card.body` if body is defined (not `card.notes`).
       - `deleteCard(data, id)`: Find parent info, splice from array, no recompact. Uses findParent to get the container array.
       - `moveCard(data, cardId, newParentId, requestedPosition)`: Splice from source array. Push to end of target array (or splice at `requestedPosition` if given). No position field. Cycle check unchanged.
       - `countDescendants`: traverse `card.children` not `card.children.cards`
       - `listCards(data, parentId)`: return getContainer(data, parentId) or [] -- just the plain array
       - `getChildren(data, id)`: return `card.children || []`
       - REMOVE: `recompact()`, `getOrderedChildren()` from code AND module.exports
       - NOTE on findParent return shape: Currently returns `{parent, container}` where container was the `{ordering, cards}` object. Now container should be the parent array. But `deleteCard` uses `container.cards.findIndex(...)` and `container.cards.splice(...)`. With the new shape where container IS the array, change to `container.findIndex(...)` and `container.splice(...)`. Similarly in moveCard.
