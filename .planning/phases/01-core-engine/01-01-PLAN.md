  <name>Task 1: Create core helpers and storage module</name>
  <files>.claude/burrow/lib/core.cjs, .claude/burrow/lib/storage.cjs, .claude/burrow/test/storage.test.cjs</files>
  <action>
Create `.claude/burrow/lib/core.cjs` with:
- `output(result)`: writes JSON to stdout, exits 0
- `errorOut(message, code)`: writes `{success: false, error: message, code}` to stdout, exits 1. Code defaults to "INVALID_OPERATION"
- `ensureDataDir(cwd)`: creates `.planning/burrow/` directory (recursive) if not exists. Does NOT create items.json (storage.load handles empty state)
- `generateId(existingIds)`: generates 8-char hex ID via `crypto.randomUUID().replace(/-/g, '').slice(0, 8)`. Checks against existingIds Set, regenerates on collision. Returns the ID string.
- `collectAllIds(data)`: recursively walks tree, returns Set of all IDs. Used by generateId for collision check.

Create `.claude/burrow/lib/storage.cjs` with:
- `load(cwd)`: reads `.planning/burrow/items.json`. If file does not exist, returns `{version: 1, ordering: "custom", items: []}`. Uses `JSON.parse(fs.readFileSync(...))`.
- `save(cwd, data)`: (1) calls ensureDataDir, (2) if items.json exists, copies to items.json.bak via `fs.copyFileSync`, (3) writes to items.json.tmp via `fs.writeFileSync`, (4) renames tmp to items.json via `fs.renameSync`. Uses `JSON.stringify(data, null, 2) + '\n'` for formatting.
- Both functions use `path.join(cwd, '.planning', 'burrow', 'items.json')` for path construction.

Create `.claude/burrow/test/storage.test.cjs` with tests using `node:test` and `node:assert`:
- Test load returns default structure when file missing
- Test load reads existing items.json correctly
- Test save creates items.json with correct content
- Test save creates .bak backup of previous file before writing
- Test save uses atomic write (verify .tmp does not remain after save)
- Use `os.tmpdir()` + `fs.mkdtempSync()` for isolated test directories (do NOT use the real .planning/burrow/)
- Clean up temp dirs in test teardown

All files use CommonJS (.cjs), `'use strict'`, and require Node.js built-ins only.
  </action>
  <verify>
    <automated>node --test .claude/burrow/test/storage.test.cjs</automated>
  </verify>

<task type="auto" tdd="true">
  <name>Task 2: Create tree module with all recursive operations</name>
  <files>.claude/burrow/lib/tree.cjs, .claude/burrow/test/tree.test.cjs</files>
  <behavior>
    - findById: finds item at any depth, returns null for missing ID
    - findParent: returns {parent, container} for any item; parent is null for root items, container is the {ordering, items} object
    - getContainer: returns root data object when parentId is null, returns item.children when parentId is valid, errors on invalid parentId
    - getPath: returns array of items from root to target (ancestry chain), null if not found
    - addItem: adds item at root (no parentId) or as child, generates 8-char hex ID, sets created timestamp, position based on container ordering (custom=end or requested, alpha=0), children defaults to {ordering:"custom", items:[]}
    - editItem: updates title and/or notes and/or ordering on existing item, returns updated item
    - deleteItem: removes item and all descendants from tree, recompacts sibling positions, returns {id, title, descendantCount}
    - moveItem: removes from source, inserts into target, recompacts source positions, assigns position by target ordering, cycle check (cannot move into own descendant), move to root valid (parentId=null)
    - getChildren: returns ordered children of an item (respecting ordering mode), returns empty array for leaf items
    - listItems: no arg returns ordered root items; with parentId returns ordered children of that item (same as getChildren but accessible by name for CLI clarity)
    - recompact: after delete/move-out, reassigns positions [0,1,2,...] to remaining siblings sorted by current position
    - getOrderedChildren: sorts items by position (custom), title asc (alpha-asc), or title desc (alpha-desc)
  </behavior>
  <action>
Write tests FIRST in `.claude/burrow/test/tree.test.cjs`, then implement in `.claude/burrow/lib/tree.cjs`.

Tests must cover (using node:test describe/it blocks):
- findById: find at root, find nested 3 levels deep, return null for missing
- getPath: path from root to deeply nested item, null for missing
- addItem: add to root (custom ordering, gets position 0), add as child, add with --position override, add to alpha-asc parent (position irrelevant, sorted by title)
- editItem: edit title, edit notes, edit ordering, error on missing ID
- deleteItem: delete leaf, delete item with children (descendants removed), positions recompacted after delete
- moveItem: move between parents, move to root, cycle detection (moving parent into child rejected), positions recompacted at source, position assigned by target ordering
- getChildren/listItems: custom ordering returns by position, alpha-asc returns alphabetical, alpha-desc returns reverse alphabetical
- ID generation: IDs are 8-char hex, timestamps are ISO 8601

tree.cjs implementation:
- `findById(data, id)` -- recursive search through data.items and all children.items
- `findParent(data, id)` -- recursive search returning {parent: item|null, container: {ordering, items}}
- `getContainer(data, parentId)` -- returns `data` when parentId is null/undefined, otherwise finds item and returns `item.children`
- `getPath(data, id)` -- recursive ancestry builder, returns array [root...target] or null
- `addItem(data, {title, parentId, notes, position})` -- find container, generate ID (use core.generateId with core.collectAllIds), create item object, assign position via ordering logic, push to container.items, return created item
- `editItem(data, id, {title, notes, ordering})` -- find by ID, update provided fields, return item. For ordering: updates item.children.ordering
- `deleteItem(data, id)` -- find parent, count descendants recursively, splice from container.items, recompact, return {id, title, descendantCount}
- `moveItem(data, itemId, newParentId, requestedPosition)` -- cycle check via getPath, remove from source (recompact), get target container, assign position, push, return item
- `getChildren(data, id)` -- find item, return getOrderedChildren(item.children)
- `listItems(data, parentId)` -- if no parentId: getOrderedChildren(data), else: getChildren(data, parentId)
- `recompact(container)` -- sort by position, reassign [0,1,2,...]
- `getOrderedChildren(container)` -- switch on ordering: custom sorts by position, alpha-asc/desc sorts by title

Use `require('./core.cjs')` for generateId and collectAllIds. Do NOT import storage -- tree.cjs is pure data manipulation.

RED-GREEN cycle: write all tests first (they will fail), then implement tree.cjs until all pass.
