Three targeted fixes:

**Fix 1: Move --to flag (burrow-tools.cjs, UAT gap #6)**
In the `case 'move'` block (line ~194), add `to` as a parseArgs option alongside `parent`:
```
options: {
  to: { type: 'string' },
  parent: { type: 'string' },
},
```
Then when computing `newParentId`, check `values.to` first, fall back to `values.parent`:
```
const rawParent = values.to !== undefined ? values.to : values.parent;
```
Use `rawParent` in the existing undefined/empty/root logic instead of `values.parent`. This makes `--to` the primary flag while keeping `--parent` for backward compatibility.

Also set `strict: true` (remove `strict: false`) so unknown flags are caught instead of silently dropped.

**Fix 2: Archive tag in --archived-only (render.cjs line 196, UAT gap #12)**
Change line 196 from:
```
const showArchivedLabel = filter === 'include-archived';
```
to:
```
const showArchivedLabel = filter === 'include-archived' || filter === 'archived-only';
```

**Fix 3: Count column alignment (render.cjs line 121, UAT gap #7)**
In `formatCardLine`, change `countStr` logic so zero-count cards still occupy space. Instead of:
```
const countStr = descCount > 0 ? `  (${descCount})` : '';
```
Always produce a count string. When descCount is 0, use empty padding of the same width as the widest count in the sibling set. Since formatCardLine is called per-card without sibling context, the simplest fix is to always show the count -- including `(0)` for leaves:
```
const countStr = `  (${descCount})`;
```
This ensures every line has a consistently-sized right column.

Two bugs to fix for --depth to work in pretty-print mode (UAT gap #13):

**Bug 1: Router filters out depth > 1 entries (burrow-tools.cjs line ~292)**
Currently the `get` command (around line 290-292) filters renderTree results to `depth === 1` only:
```
const childCards = treeResult.cards.filter((c) => c.depth === 1);
```
This discards all depth 2+ entries. Instead, reconstruct a nested tree from the flat renderTree output. Write a helper function `nestFlatCards(flatCards)` that:
1. Takes the flat array from renderTree (each entry has `id`, `depth`, `children` array, etc.)
2. Uses a stack-based approach: iterate flat cards in order, for each card at depth N, attach it as a child of the most recent card at depth N-1
3. Returns only the depth-1 cards, but now with their deeper descendants nested in `.children`

Replace the filter line with:
```
const childCards = nestFlatCards(treeResult.cards.filter(c => c.depth >= 1));
```

Place `nestFlatCards` as a local function near the top of burrow-tools.cjs (not exported, just a helper).

**Bug 2: renderCard has no recursive rendering (render.cjs lines 197-200)**
Currently renderCard's children section is a flat loop calling formatCardLine once per child. It does not recurse into grandchildren. Add a recursive helper `renderTreeLines(children, depth, indent, tw, showArchived)` that:
1. Iterates children, for each child calls formatCardLine with appropriate prefix
2. If child has `children` array with entries, recurses with increased indent (pipe character for non-last items, spaces for last)
3. Uses the same box-drawing characters: BRANCH for middle, CORNER for last, PIPE for continuation

Replace the flat loop in renderCard (lines 197-200) with a call to this recursive helper. The indentation pattern for nested levels:
- Level 0: `├── ` / `└── `
- Level 1: `│   ├── ` / `│   └── ` (or `    ├── ` / `    └── ` if parent was last)

The recursive helper should return an array of formatted lines.
